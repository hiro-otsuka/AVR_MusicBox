＝＝＝＝ MMLの仕様 ＝＝＝＝ 
■パート共通定義
 Tnnn ... テンポ設定
  ただし通常のテンポ（四分音符の個数等）とは異なり、一分音符の長さを設定する。
  計算式は以下の通り。
   ((60 / 四分音符のリズム数) * 4) / ((1,024 * 256) / CPUの周波数) 
      ⇒計算結果から 3*16 の倍数の一番近い数字に丸める。
        ただし、3連符を使わない場合は 16 の倍数でもよい。
  CPUの周波数が16MHzの時、 (60/四分音符のリズム数)*4 / 0.016384 に等しい
    例）四分音符=120の楽譜で、CPU=16MHz の場合
     ((60/120)*4)/0.016384 = 122
      ⇒ 3連符を使う場合は 96 or 144、使わない場合は 128 に丸める
 Bnn/mm ... 1小節の音符数（拍子）
  各パートの小節チェックで使用される。
 K{ xxx } ... キー設定
  キーを設定する。例えば以下の通り。
    E minor ... K{ F+ }
    A flat major ... K{ A-B-D-E- }
  ただし各音符でも指定できるので、必須ではない。

■パート定義
 パートは #{ xxxx } で定義する。
 ツールの仕様上はパート数の制限はないが、3パートまでのファイルのみ再生可能。

■パート内のコマンド（制御系）
 Knn ... 変調を指定。
 Vnn ... ボリュームを指定。0-16が指定可能。
 @nn ... 波形の種類を指定。0=サイン波、1=三角波、2=矩形波
 W(nn,mm) ... 音の減衰傾斜を指定する。
  最初の nn で減衰までの時間、次の mm で減衰間隔を指定する。
   CPUの周波数が 16MHz の時で、
    nn = 1 の場合、減衰まで 0.016384秒。
    mm = 1 の場合、0.016384秒ごとにボリューム -1。
   それぞれ設定値に比例して時間（秒数）が伸びる。
 | ... 小節の区切りを表す。音符数と拍子が不一致の場合は警告を表示する。
 : ... 小節の区切りを表すが、音符数と拍子が不一致の場合も警告を表示しない。

■パート内のコマンド（音符系）
 Lnn ... デフォルトの音符長を指定。
 Onn ... オクターブを指定。0-4 が指定可能。
 > ... オクターブ+1
 < ... オクターブ-1
 [a-g](移調指定)(nn) ... 音符の指定
  c,d,e,f,g,a,b ... ド～シの音符
  (移調指定) ... +, -, = の指定が可能。= はキー設定を打ち消す場合に使用。
  nn ... 音符の長さ。四分音符の場合は 4 を指定。省略すると L の設定値となる。
 ^ ... タイ。音符の長さをつなげるコマンド。
  四分音符を2個つなげる場合、
    a4^4
  とする。二分音符と同じ長さになる。小節をまたがってつなげる場合、
    a4^|4
  のように指定することで拍子の確認が正確に行われる。
